# Contributing to CADMIES

First off, thank you for considering contributing to CADMIES! 🧑🏽‍🌾

CADMIES is part of **Project Hierion** — a sacred place for interconnected
knowledge. The mycelium grows through the efforts of everyone who tends it.

## Code of Conduct

This project adheres to the principles of ethical knowledge sharing
(AGPLv3 with Commons Clause). We expect all contributors to:

- Respect the Commons Clause (no commercial exploitation without reciprocity)
- Share improvements back to the community
- Prioritize educational and research purposes
- Be excellent to each other

## How to Contribute

### 1. Find Something to Work On

- Check the [Issues](https://github.com/Project-Hierion/Hierion-CADMIES/issues) tab
- Look for issues tagged `good first issue` or `help wanted`
- Or open a new issue to propose something

### 2. Claim It

Comment on the issue: "I'm working on this."

### 3. Fork and Branch

Fork the repo, then create a branch:
```
git checkout -b feature/your-feature-name
```


### 4. Make Changes

- Follow PEP 8 for Python
- Use descriptive variable names
- Add docstrings for functions
- Type hints encouraged
- Test your changes
- Update documentation if needed

### 5. Commit and Push
```
git add .
git commit -m "Add: brief description of what you did"
git push origin feature/your-feature-name
```

### 6. Open a Pull Request

Go to GitHub, click "Compare & pull request," describe your changes.

## Style Guide

### Python
- Follow PEP 8
- snake_case for human_ids
- Use docstrings

### JSON Schemas
- Follow the CID specification
- No embedded domain/type in human_id

All contributors will be:
- Added to CONTRIBUTORS-GARDENERS.md
- Mentioned in release notes

We also credit the repositories, projects, tools, and platforms that
CADMIES builds upon — even those that only provided inspiration. The
hyphen is a handshake. Every piece of the stack that made the mycelium
possible deserves acknowledgment.
